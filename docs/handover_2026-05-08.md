# Handover: Multi-Venv Architecture (2026-05-08)

## Session Summary

Long troubleshooting session focused on integrating CosyVoice 3 into nSpeech. The session revealed fundamental dependency conflicts between TTS engines and led to an architectural decision to isolate each engine in its own venv.

## What Happened

### CosyVoice 3 Integration Journey

1. **Started with a handover** noting CosyVoice adapter was complete but blocked by dtype mismatch (transformers 5.x loads safetensors as BFloat16, llm.pt is Float32).

2. **Applied `.llm.float()` fix** - resolved dtype crash but audio was gibberish (short Chinese-sounding syllables).

3. **Root cause analysis** - Discovered CosyVoice3 has **zero built-in voices**. The bundled `zero_shot_prompt.wav` is a Chinese reference audio. Using it for English text produces Chinese-accented gibberish.

4. **Cloning works** - Cloned an English voice (Allan Watts, 34s audio clipped to 30s). Cloning produced valid embeddings, but generation still produced short blips (~0.12s audio).

5. **GitHub issue research** - Found issue #1446: `transformers==4.53.1` causes garbled speech, `4.51.3` works. We had 5.8.0.

6. **Downgraded to transformers 4.51.3** - This fixed the model loading but `inference_zero_shot` with `zero_shot_spk_id` still produced tiny audio.

7. **Key discovery: `inference_instruct2` works** - The only inference mode that produces proper audio with cloned voices is `inference_instruct2` with `zero_shot_spk_id`. All other modes fail:
   - `inference_zero_shot` with `zero_shot_spk_id` → ~0.12s audio
   - `inference_cross_lingual` → assertion error, strips `<|endofprompt|>`
   - `inference_instruct2` with `zero_shot_spk_id` → **works, 3-7s proper audio**

8. **CosyVoice generated proper English speech** with the Allan Watts clone.

### The Dependency Conflict

9. **Chatterbox broke** - User tried to use Chatterbox, which wasn't installed. Installing Chatterbox (`pip install -r requirements/chatterbox.txt`) upgraded `transformers` to 5.2.0, breaking CosyVoice.

10. **Attempted fixes** - Tried `.llm.float()` with transformers 5.x, but CosyVoice still had issues. Tried pinning versions back but pip had access denied errors on some packages.

11. **Venv became corrupted** - Mixed dependencies caused: 30+ second startup, short audio blips, Ctrl+C hanging. Unrecoverable state.

### Architectural Decision

12. **Rolled back to last working commit** (Kokoro + Chatterbox working, no CosyVoice).

13. **Decided on separate venvs per engine** - This is the only clean solution given:
    - CosyVoice needs `transformers==4.51.3`
    - Chatterbox/Kokoro need `transformers>=5.0`
    - These are fundamentally incompatible in one venv

## Multi-Venv Architecture Plan

### Design
Each engine gets its own venv with isolated dependencies. Same codebase shared across all venvs.

### Install
```bash
python install.py install --engine kokoro      # Creates venv-kokoro
python install.py install --engine cosyvoice   # Creates venv-cosyvoice  
python install.py install --engine chatterbox  # Creates venv-chatterbox
python install.py install --engine all         # Creates all three
```

### Run
```bash
venv-kokoro\Scripts\python run.py       # Kokoro on port 8000
venv-cosyvoice\Scripts\python run.py    # CosyVoice on port 8000
```

### Deployment
| Host | Engine | Hardware | Notes |
|------|--------|----------|-------|
| FATTEN | Kokoro | Intel GPU | English base voice service |
| BADKID | CosyVoice3 | RTX 4090 | Multilingual, shares VRAM with Qwen3.6 (80k ctx) |
| — | Chatterbox | — | Archived, English-only |

### How it works
- `src/` and `run.py` are shared code
- Each venv has its own Python with engine-specific dependencies
- Engine adapters in `src/nspeech/engines/` import engine packages that only exist in their respective venvs

## Key CosyVoice Facts (for next integration)

- **Dependencies**: `transformers==4.51.3`, `tokenizers==0.21.0`, `huggingface-hub==0.30.0`
- **Environment**: `NUMBA_DISABLE_JIT=1` required
- **Python paths**: Must add `models/third_party/CosyVoice` and `models/third_party/CosyVoice/third_party/Matcha-TTS`
- **Inference**: Only `inference_instruct2` with `zero_shot_spk_id` works
- **Token**: All text needs `<|endofprompt|>` suffix
- **Cloning**: 30s max audio, clip before cloning
- **No built-in voices**: Must clone a reference audio first
- **Voice cache load**: Use `torch.load(path, weights_only=False, map_location='cpu')`
